Tags & Element Syntax

Understanding the Building Blocks

In HTML, we use tags to tell the browser how to display content. When you combine tags with the content inside them, you create an element. Think of tags as the "instructions" and the element as the "final product."

The Sandwich Analogy: The opening and closing tags are like the two slices of bread. The content inside is the filling. You need all three parts to have a complete sandwich (an element).

Basic Syntax Anatomy

Every standard HTML element follows a specific pattern. It starts with an opening tag, contains content, and ends with a closing tag (marked by a forward slash).

<!DOCTYPE html>
<html>
  <head>
    <title>My Web Page Title</title>
  </head>
  <body>
    /* This is where the main content of your webpage will go */
    <h1>This is a Heading</h1>
    <p>Hello World!  This is a paragraph.</p>
  </body>
</html>

Term What it is Example
Tag The individual markers (angle brackets). <p> or </p>
Element The whole unit from start to finish. <p>Hello</p>
Content The text or data inside the tags. "Hello"

Key Takeaways

  • Always Close: Most tags need a closing tag </tag> or the browser will get confused.
  • Case Sensitivity: While not strictly required, always write tags in lowercase for modern web standards.
  • Nesting: Elements can go inside other elements (like a box inside a box), but you must close them in the order they were opened.
